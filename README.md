Grade Clicker

Название проекта
Grade Clicker — интерактивное приложение-кликер для симуляции набора баллов по курсу «Разработка мобильных приложений». Пользователь нажимает на центральное изображение, за каждый клик начисляется 5 баллов. При достижении определённых порогов (50, 70, 90, 100) автоматически меняются отображаемая оценка, цвет интерфейса и фоновое изображение. Проект создан для наглядной демонстрации работы жизненного цикла Activity в Android.

Цель работы
Изучить жизненный цикл Activity (Activity Lifecycle) в платформе Android, научиться корректно обрабатывать переходы между состояниями с помощью callback-методов, а также освоить механизмы сохранения состояния UI при изменении конфигурации устройства (например, повороте экрана) с использованием Jetpack Compose и rememberSaveable.

Ответы на контрольные вопросы
1. Какие три метода вызываются при запуске приложения?
   При запуске Activity последовательно вызываются: onCreate() → onStart() → onResume().

2. Чем отличается onPause() от onStop()?
   onPause() вызывается, когда Activity теряет фокус ввода, но остаётся частично видимой (например, поверх появилось диалоговое окно или прозрачное Activity). onStop() вызывается, когда Activity становится полностью невидимой для пользователя (например, при нажатии кнопки «Домой» или переходе в другое приложение).

3. Когда вызывается onRestart()?
   Вызывается только в том случае, если Activity ранее перешла в состояние Stopped (onStop()), но не была уничтожена, и теперь пользователь возвращает её на передний план. Всегда выполняется перед onStart().

4. Что происходит при повороте экрана?
   Поворот экрана является изменением конфигурации (configuration change). По умолчанию Android полностью уничтожает текущую Activity (onDestroy()), а затем создаёт её заново (onCreate()) с учётом новой ориентации и ресурсов. Если данные не сохранены явно, они теряются.

5. В чём разница между remember и rememberSaveable?
   remember сохраняет значение только в рамках текущей рекомпозиции Compose, но сбрасывает его при пересоздании Activity. rememberSaveable сохраняет состояние не только при рекомпозиции, но и при изменении конфигурации (поворот, смена языка), используя встроенный механизм сохранения состояния Android (Bundle).

6. Почему onCreate() должен вызывать super.onCreate()?
   Вызов super.onCreate(savedInstanceState) необходим для того, чтобы базовый класс Activity выполнил свою внутреннюю инициализацию: привязал контекст, восстановил ранее сохранённое состояние и подготовил инфраструктуру жизненного цикла. Без этого вызова приложение может работать некорректно или аварийно завершаться.

7. Для чего используется класс Log?
   Класс android.util.Log используется для вывода отладочных сообщений в системную консоль Logcat. Он позволяет разработчику отслеживать выполнение кода, фиксировать ошибки (Log.e()), предупреждения (Log.w()), информационные события (Log.i()) и подробные отладочные данные (Log.d()).

8. Какой метод вызывается только один раз за время жизни Activity?
   onCreate(). Он вызывается единожды при первоначальном создании экземпляра Activity и служит точкой входа для инициализации UI и данных.
